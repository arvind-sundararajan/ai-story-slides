Introduction

BAML was created to solve the problem of reliable tool-calling with any model.

BAML, or BoundaryML, is a simple prompting language designed to build reliable AI workflows and agents. It achieves this by turning prompt engineering into schema engineering, allowing users to focus on the models of their prompt to get more reliable outputs. This approach enables users to create agents and workflows using chained BAML functions, with support for streaming and retries, without requiring them to write their entire application in the language. Instead, users only need to write the prompts in BAML, which can then be wired up with LLM functions in any language of choice, including Python, TypeScript, Ruby, and Go.

The core principle of BAML is that LLM prompts are functions, with every prompt being a function that takes in parameters and returns a type. This core principle is the foundation of BAML's architecture, which is based on functions, with each function defining which models it uses and what its prompt is. BAML functions can be defined using a simple syntax, with support for parameters, returns, and model specification. The language also provides support for classes, such as Message, ReplyTool, and StopTool, which can be used to define the structure of prompts and responses.

Additionally, BAML comes with full typesafety, streaming, retries, and wide model support, even when models do not support native tool-calling APIs.

BAML provides a range of benefits, including fast iteration speed, allowing users to test 240 ideas in the same amount of time it would take to test 10 ideas with a slower pipeline. The language also provides support for production use cases, with examples of interactive app examples and prompt fiddle available. Furthermore, BAML enables users to test prompts 10 times faster, right in their IDE, and reduces the time it takes to test pipeline ideas from 2 minutes to 5 seconds. This is achieved through native tooling for VSCode, with support for JetBrains and Neovim coming soon, allowing for visualization of full prompts and API requests.

This book is designed for developers and users who want to learn how to build reliable AI workflows and agents using BAML. Through this book, readers will learn how to use BAML to create agents and workflows, how to define BAML functions, and how to integrate BAML with other languages and tools. They will also learn about the benefits of using BAML, including fast iteration speed, support for production use cases, and native tooling for popular IDEs. By the end of this book, readers will have a deep understanding of BAML and how to use it to build reliable AI workflows and agents.
